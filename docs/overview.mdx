TraceGuard is a **local-first** developer tool that records what AI coding agents
did to your project. It is a black box recorder, safety layer, cost tracker, and
patch-review dashboard for tools like Claude Code, Cursor, Copilot, and others.

TraceGuard tells you:

- **what changed** — files created, modified, or deleted
- **what ran** — the commands executed during a session
- **what looked risky** — blocked/warned commands and protected files
- **what may have leaked** — secrets detected in diffs, output, or files
- **what it cost** — API/model usage when it is available
- **what broke** — build/test results and a deterministic failure diagnosis
- **how to undo it** — Git-based checkpoints and rollback

## What TraceGuard is not

- It is **not** a chatbot or an AI coding assistant.
- It does **not** replace Claude Code, Cursor, or Copilot.
- It does **not** send your project data anywhere. Everything stays on your
  machine in `~/.traceguard` and your project's `.traceguard/` folder.

TraceGuard *wraps, observes, records, guards, and reviews* the agents you already
use.

## How it works

You launch an agent or command through the wrapper:

```bash
trg run "claude fix the login bug"
```

TraceGuard creates a checkpoint, captures the starting Git state, watches the
project for file changes, records command output, scans for secrets and risky
commands, captures the final Git diff, optionally runs your configured checks,
and stores everything in a local SQLite database. You review it all in a local
dashboard at `http://127.0.0.1:<port>`.

<Note>
TraceGuard observes terminal-based agents most directly. For GUI-based tools it
can still watch file changes and Git diffs, but it cannot guard the tool's
internal actions. See [Limitations](/limitations).
</Note>
