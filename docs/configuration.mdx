---
title: Configuration
description: The per-project .trace/config.toml file.
---

`trace init` creates `.trace/config.toml` in your project. Configuration is
intentionally small.

```toml
# Trace project configuration

project_name = "My App"
protected_files = [".env", ".env.local", "id_rsa", "secrets.json"]
default_checks = ["npm run build", "npm test"]
created_at = "2026-06-14T20:30:00Z"
```

| Field | Description |
| --- | --- |
| `project_name` | Display name shown in the dashboard. |
| `protected_files` | File names/paths that raise a warning when changed during a run. Matched by file name or path suffix. |
| `default_checks` | Commands run after a run completes (build/test). Results appear as test results and timeline events. |
| `created_at` | When the project was initialized. |

`project_name` is the only field settable via the CLI: `trace config set
project_name "My App"`. See [CLI Commands](/cli-commands).

## Checks

When `default_checks` is set, Trace runs them after each `trace run` and
records pass/fail results. Skip them for a single run with:

```bash
trace run "claude" --no-checks
```

## Storage locations

| Path | Contents |
| --- | --- |
| `~/.trace/trace.db` | The global SQLite database. |
| `~/.trace/daemon.json` | Daemon PID, port, and start time. |
| `~/.trace/bin/trace` | Installed binary (via the install script). |
| `<project>/.trace/config.toml` | Per-project config. |
| `<project>/.trace/runs/<run_id>/` | Captured stdout/stderr logs. |

All of this stays on your machine.
