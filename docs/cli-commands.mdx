The binary is `trace`.

## `trace init`

Initialize Trace in the current project. Creates `.trace/config.toml`,
registers the project via the daemon in `~/.trace/trace.db`, and warns if the
folder is not a Git repository. Trace never initializes Git for you.

```bash
trace init
```

## `trace run <command>`

Run a command under Trace monitoring.

```bash
trace run claude
trace run "claude fix the bug"
trace run "npm test"
trace run "cursor ."
```

| Flag | Description |
| --- | --- |
| `--no-checks` | Skip configured project checks for this run. |
| `-y`, `--yes` | Auto-approve commands that would otherwise prompt. |

The run lifecycle: detect project → create run → checkpoint Git → start file
watcher → execute command (output is teed to the console and logs) → capture
final Git diff → scan secrets → run configured checks → store everything.

## `trace dashboard`

Start the daemon if needed, find the active port, and open the dashboard in
your browser at `http://127.0.0.1:<port>`.

```bash
trace dashboard
```

## `trace doctor`

Run system checks: toolchain (git, node, npm), clipboard availability, daemon
status, key paths (database, install directory, whether it's on `PATH`), and
which AI CLI tools are detected on your machine.

```bash
trace doctor
```

## `trace scan`

Detect the current project's stack and print it: package manager, languages,
frameworks, TypeScript usage, test framework, scripts, README presence, env
files, deployment config, and Git branch/dirty status.

```bash
trace scan
```

## `trace runs`

List recent runs across your projects (status, files changed, secret
warnings, command, run id).

```bash
trace runs
```

## `trace show <run_id>`

Show a run's summary and timeline: command, project, status, exit code,
start/end time, files/commands/secrets counts, estimated cost (when known),
and the full event timeline.

```bash
trace show <run_id>
```

## `trace replay <run_id>`

Replay a run's events, commands, and file changes in the order they actually
happened, paced by their real recorded timestamps (clamped between 150ms and
1.5s per step so a long run never drags). Reads the same data as `show`,
`patch`, and `risks` — nothing is simulated.

```bash
trace replay <run_id>
trace replay <run_id> --fast   # skip pacing, print immediately
```

## `trace patch <run_id>`

Show the changed files for a run, derived from the Git diff.

```bash
trace patch <run_id>
```

## `trace risks <run_id>`

Show guarded commands (warned/blocked/require-approval) and redacted secret
warnings for a run.

```bash
trace risks <run_id>
```

## `trace costs <run_id>`

Show API usage and estimated cost for a run (provider, model, input/output
tokens, estimated cost, and a total).

```bash
trace costs <run_id>
```

## `trace checkpoints`

List checkpoints with a Git ref across recent runs, for use with rollback.

```bash
trace checkpoints
```

## `trace config show` / `trace config set <key> <value>`

Show or change the current project's configuration.

```bash
trace config show
trace config set project_name "My App"
```

Currently the only settable key is `project_name`.

## `trace integrations` / `trace integrations status`

List integration surfaces (Claude Code, Codex CLI, Cursor, VS Code, GitHub),
or check what is live right now (daemon running, GitHub token available).

```bash
trace integrations
trace integrations status
```

## `trace rollback`

Restore the most recent checkpoint, or pick one if several exist. Rollback is
Git-based and asks for confirmation. Use `-y`/`--yes` to skip the prompt.

```bash
trace rollback
trace rollback -y
```

## `trace update`

Update the `trace` binary to the latest GitHub release. Homebrew installs are
directed to `brew upgrade trace` instead.

```bash
trace update
```

## `trace github <status | commits | pulls | cat>`

Read directly from the project's GitHub repo — **including private repos** —
using a token from `GITHUB_TOKEN`/`GH_TOKEN`, the `gh` CLI, or
`~/.trace/github.json`. Read-only.

```bash
trace github status                 # auth + repo connection
trace github commits --limit 10     # recent commits from the remote
trace github pulls                  # open pull requests
trace github cat src/main.rs        # print a file from the remote repo
trace github cat README.md --ref some-branch
```

## `trace daemon start | stop | status`

Manage the local daemon.

```bash
trace daemon start    # start the local server
trace daemon stop     # stop it
trace daemon status   # running?, port, database path, active projects
```

## `trace --help` / `trace --version`

Standard help and version output (`trace --version` prints `Trace 1.2`).
