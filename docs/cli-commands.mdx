The main command is `trg`. `traceguard` is a long alias for the same binary.

## `trg init`

Initialize TraceGuard in the current project. Creates `.traceguard/config.toml`,
registers the project in `~/.traceguard/traceguard.db`, and warns if the folder
is not a Git repository. TraceGuard never initializes Git for you.

## `trg run <command>`

Run a command under TraceGuard monitoring.

```bash
trg run claude
trg run "claude fix the bug"
trg run "npm test"
trg run "cursor ."
```

| Flag | Description |
| --- | --- |
| `--no-checks` | Skip configured project checks for this run. |
| `-y`, `--yes` | Auto-approve commands that would otherwise prompt. |

The run lifecycle: detect project → create run → checkpoint Git → start file
watcher → execute command (output is teed to the console and logs) → capture
final Git diff → scan secrets → run configured checks → store everything.

## `trg dashboard`

Start the daemon if needed, find the active port, and open the dashboard in your
browser at `http://127.0.0.1:<port>`.

## `trg rollback`

Restore the most recent checkpoint, or pick one if several exist. Rollback is
Git-based and asks for confirmation. Use `-y`/`--yes` to skip the prompt.

## `trg compress-prompt [text]`

Compress a prompt locally with TraceCompress. Omit the text to paste
interactively. See [Prompt Compression](/prompt-compression).

| Flag | Description |
| --- | --- |
| `--mode <m>` | `normal` \| `concise` \| `bare`. Defaults to project config. |
| `--output-budget <p>` | Append a budget block: `tiny` \| `short` \| `normal` \| `detailed`. |
| `-y`, `--yes` | Accept without prompting. |

## `trg run --compress <command>`

Compress the agent prompt before running. Shows before/after and estimates, asks
for confirmation, then runs with the compressed prompt and stores the metadata
with the run. Use `--mode` to pick the compression mode.

```bash
trg run --compress "claude fix the login bug but do not rewrite the whole app"
trg run --compress --mode bare "claude add tests for src/auth/login.ts"
```

## `trg update`

Update the `trg` binary to the latest GitHub release. Homebrew installs are
directed to `brew upgrade traceguard` instead.

## `trg github <status | commits | pulls | cat>`

Read directly from the project's GitHub repo — **including private repos** —
using a token from `GITHUB_TOKEN`/`GH_TOKEN`, the `gh` CLI, or
`~/.traceguard/github.json`. Read-only.

```bash
trg github status                 # auth + repo connection
trg github commits --limit 10     # recent commits from the remote
trg github pulls                  # open pull requests
trg github cat src/main.rs        # print a file from the remote repo
trg github cat README.md --ref some-branch
```

## `trg daemon start | stop | status`

Manage the local daemon.

```bash
trg daemon start    # start the local server
trg daemon stop     # stop it
trg daemon status   # running?, port, database path, active projects
```

## `trg --help` / `trg --version`

Standard help and version output.
