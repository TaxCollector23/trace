---
title: Secret Detection
description: Local, rule-based secret scanning that never stores raw secrets.
---

TraceGuard scans for secrets locally using a rule-based scanner. It scans:

- changed files
- the Git diff
- command output
- environment-related and protected files

## What it detects

- `.env`-style and other protected files
- OpenAI, Anthropic, and Google Gemini API keys
- Groq API keys
- GitHub tokens
- AWS access keys
- private SSH keys
- JWTs
- database URLs
- generic API keys and bearer tokens

## Redaction

TraceGuard **never stores raw secrets**. Every detected value is redacted to a
short prefix before it is stored or displayed:

```text
sk-ant-...redacted
ghp_...redacted
AKIA...redacted
```

Each finding stores the secret type, file path (when applicable), the redacted
value, the action taken, the run ID, and a timestamp. View them in the
**Risk Center**.

## Protected files

Files listed in `protected_files` in your project config (e.g. `.env`,
`id_rsa`, `secrets.json`) raise a warning when they are changed during a run,
even if no secret pattern matches their contents.
