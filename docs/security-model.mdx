---
title: Security Model
description: How TraceGuard keeps your data local and safe.
---

TraceGuard is local-first and conservative by design.

## Network

- The daemon binds to **`127.0.0.1` only** — never `0.0.0.0`, never the local
  network. The dashboard is reachable only from your machine.
- There is no account and no authentication for the MVP, because there is no
  remotely reachable surface.

## Secrets

- Raw secrets are **never stored**. Detected secrets are redacted to a short
  prefix plus `...redacted` before storage or display.
- Protected files (e.g. `.env`, `id_rsa`, `secrets.json`) raise warnings when
  changed.

## Uploads

- No cloud upload by default. Project data stays on your machine.
- The GitHub integration uploads **only sanitized summaries** — never raw
  project files, raw secrets, or the local SQLite database.
- The Vercel landing page and the optional Firebase mirror are static and must
  **not** connect to the local daemon.

## Prompt Compressor

- Compression is local and deterministic by default; prompt text does not leave
  the machine.
- Prompt text is only stored when `prompt_history = true` (default; disable to
  store stats only).
- Any future LLM-based compression is opt-in and clearly labelled, because
  prompt text would then leave the machine.

## Git & rollback

- Rollback is Git-based and always confirmed. Checkpoints use `git stash create`
  (capturing dirty state without disturbing the working tree) or `HEAD`.

## Data locations

| Path | Contents |
| --- | --- |
| `~/.traceguard/traceguard.db` | Local SQLite history |
| `~/.traceguard/daemon.json` | Daemon pid/port/start time |
| `<project>/.traceguard/` | Per-project config and run logs |
