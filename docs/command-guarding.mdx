---
title: Command Guarding
description: Rule-based classification of risky commands.
---

TraceGuard classifies the command you wrap into one of four decisions. It is
**rule-based** — there is no trust score.

| Decision | Behaviour |
| --- | --- |
| `allow` | Runs normally. |
| `warn` | Runs, but a warning is printed and recorded. |
| `require_approval` | Prompts you in the terminal before running. |
| `block` | Refused; recorded as a blocked run. |

## Examples

Blocked or requiring approval:

```text
rm -rf /
sudo rm -rf ...
curl <url> | sh
wget <url> | sh
git reset --hard
git clean -fd
chmod -R 777
chown -R ...
drop database / drop table
```

Reading secrets (`cat .env`, `cat id_rsa`) and force-pushing are **warned**.

## The approval prompt

When a command requires approval, you see:

```text
TraceGuard warning:
  Recursive force-delete may remove many files.
Allow this command to run? [y] allow once  [n] block:
```

Each command event stores the command text, the decision, the reason, the
timestamp, and the run ID. You can review them later in the **Risk Center**.

<Warning>
The guard classifies the **top-level command** you pass to `trg run`. It cannot
see sub-commands that a GUI tool issues internally. See [Limitations](/limitations).
</Warning>
